## 产品概述

对现有总运营后台（Platform）进行整体功能重构，使其与业务模式对齐：平台作为游戏平台供应商，向店铺销售游戏豆，不参与店铺具体运营。

## 核心需求

1. **移除店铺运营工具**：运营后台不应管理优惠券、活动配置，需从菜单和路由中移除
2. **新增游戏豆销售管理**：平台核心收入模块，需独立可视化各店铺购买趋势、余额预警
3. **改造数据中心**：大屏看板增加平台核心指标；数据报表增加内容消费、设备运行、订单流水、跨店会员等全局视角
4. **保留原有模块**：门店管理、内容中心、用户体系、平台财务、系统运维、运维支持
5. **预留代理商扩展**：架构上预留商家-门店多层级数据隔离能力，为后续代理商机制做准备

## Tech Stack Selection

- 前端框架：Vue 3 + TypeScript + Vite（复用现有项目技术栈）
- UI 组件库：Naive UI（与现有页面完全一致）
- 图表库：ECharts（Dashboard/Reports 已有使用，复用）
- 路由：Vue Router 4（复用现有配置）
- 状态管理：Composables / 组合式函数（与现有风格一致）

## Implementation Approach

采用"菜单骨架先行，页面逐个填充"的策略：

1. 先调整 `PlatformLayout.vue` 菜单结构和 `router/index.ts` 路由，搭好新骨架
2. 基于现有 `Dashboard.vue` 和 `Reports.vue` 的指标卡片、ECharts 图表模式，改造和扩展数据视角
3. 新建页面复用已有的统计卡片、表格、筛选抽屉等组件模式，保持视觉一致性
4. 预留数据范围隔离：所有表格增加 `merchantId/storeId` 字段占位，后续对接代理商权限时可直接扩展

## Implementation Notes

- **Grounded**：复用 `PlatformLayout.vue` 的 `menuOptions` 结构、`router/index.ts` 的子路由模式、`Dashboard.vue` 的 `metrics-grid` 样式和 ECharts 图表封装
- **Performance**：大盘页面使用懒加载图表组件，避免一次性渲染过多图表造成阻塞
- **Blast radius control**：原 `Coupons.vue` / `Activities.vue` 文件保留不删除，仅从菜单和路由中移除，防止其他引用报错；新增路由使用 `() => import()` 懒加载减少首屏体积
- **Logging**：沿用现有 `console.log` 调试模式，不引入新日志库

## 设计方向

延续现有运营后台的 B 端简洁风格，以白色卡片为容器、蓝色渐变为主色调、ECharts 图表为数据可视化载体。新建页面与现有 Dashboard/Reports 保持完全一致的信息密度和交互模式。

## 页面结构（Block 设计）

### 大屏看板 (Dashboard)

- **顶部筛选栏**：日期范围选择器 + 导出 + 刷新
- **核心指标卡片行**（4-6 个）：游戏豆销售额、设备在线率、活跃会员数、内容点播次数、订单流水总额、新增门店数。每个卡片包含图标、数值、环比趋势标签。
- **主图表区**（左右两栏）：左栏"游戏豆销售趋势"折线图，右栏"内容消费 TOP10"横向柱状图
- **次图表区**（左右两栏）：左栏"设备在线率趋势"面积图，右栏"会员增长趋势"折线图

### 数据报表 (Reports)

- **顶部筛选栏**：日期范围 + 门店选择 + 导出
- **汇总指标卡片**：与 Dashboard 同风格
- **报表列表区**：按维度分类的报表入口卡片（内容消费、设备运行、订单流水、会员分析），点击下钻到对应页面

### 游戏豆销售管理 (GameBeanSales)

- **顶部筛选栏**：日期范围 + 门店选择 + 账单类型
- **汇总卡片**：总销售额、本月销售、余额预警门店数、平均客单价
- **销售趋势图**：折线图展示时间维度销售趋势
- **店铺余额表格**：店铺名称、当前余额、最近购买时间、预警状态

### 内容消费大盘 (ContentConsumption)

- **顶部筛选栏**：日期范围 + 门店 + 内容类型
- **汇总卡片**：总点播次数、最受欢迎游戏、最受欢迎影片、人均消费时长
- **内容排行图**：游戏/影片点播量 TOP10 横向柱状图
- **时段分布图**：24 小时热力分布
- **内容消费表格**：内容名称、点播次数、覆盖门店数、平均时长

### 设备运行总览 (DeviceOverview)

- **顶部筛选栏**：日期范围 + 门店 + 设备状态
- **汇总卡片**：设备总数、在线数、离线数、故障率
- **在线率趋势图**：折线图
- **设备利用率排行**：各门店利用率柱状图
- **设备状态表格**：设备编号、所属门店、状态、最近心跳时间、利用率

### 订单流水透视 (OrderFlow)

- **顶部筛选栏**：日期范围 + 门店 + 订单类型
- **汇总卡片**：总流水、订单数、客单价、退款率
- **流水趋势图**：折线图
- **门店营收排行**：柱状图
- **订单流水表格**：门店、订单号、类型、金额、时间（只读，无操作按钮）

### 跨店会员中心 (MemberCenter)

- **顶部筛选栏**：日期范围 + 商家 + 会员等级
- **汇总卡片**：会员总数、新增会员、活跃会员、储值总额
- **会员增长趋势图**：折线图
- **储值分布图**：饼图/环形图
- **会员表格**：会员信息、所属门店、储值余额、游戏币余额、最近消费时间

- **code-explorer**
- Purpose: 在调整菜单结构和路由时，快速定位所有与 PlatformLayout 和 router 相关的引用点，避免遗漏
- Expected outcome: 确认所有菜单 key、路由 path、面包屑、权限点的完整引用链

### Skill

- Purpose: 为新建页面（GameBeanSales、ContentConsumption、DeviceOverview、OrderFlow、MemberCenter）生成符合现有设计规范的高质量 Vue 组件代码，保持与 Dashboard/Reports 一致的视觉风格
- Expected outcome: 产出视觉一致、代码规范、可直接集成到项目中的页面组件
