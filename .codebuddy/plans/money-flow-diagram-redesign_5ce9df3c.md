## 用户需求

重新设计资金流通全景图，用清晰直观的方式展示头号空间平台 4 方角色之间的 5 条资金流线。

## 产品概述

一份单页 HTML 文件，用纯 SVG 手绘全景图。平台居中为核心枢纽，C端用户、商家/门店、代理商、银行账户四角色环绕四周，5 条彩色资金流线连接各方，配简短文字标注，一眼看清全貌。

## 核心特性

- 平台居中 Hub-Spoke 布局，所有角色一览无余
- 5 条资金流线用 5 种颜色区分，粗箭头 + 方向标注
- 纯 SVG 手绘，无 Mermaid 依赖，无外部 JS 库
- 浅色背景 + 高对比度，适合打印和白板讨论
- 每个角色卡片含简短职责说明
- 底部附 5 条流线的图例

### SVG 绘制要点

**角色节点**：

- 圆角矩形卡片，宽度约 200px，高度约 100px
- 顶部色条标识角色类型
- 角色名称 + 副标题（如 "C端用户 / 游客·会员"）
- 简短职责说明（如 "到店消费 VR 体验"）

**资金流线**：

- 使用 `<path>` 绘制贝塞尔曲线箭头
- 线宽 3-4px，末端带箭头标记（`<marker>`）
- 流线上方/侧方放置编号圆标 + 简短说明文字
- 5 条线颜色：
- ① 消费付款：`#3B82F6`（蓝）
- ② 游戏豆采购：`#10B981`（绿）
- ③ 代理分润：`#F59E0B`（琥珀）
- ④ 商家提现：`#06B6D4`（青）
- ⑤ 退款回流：`#EF4444`（红），虚线

**页面尺寸**：

- SVG viewBox 约 1000x800，适应主流屏幕
- 页面自适应居中，max-width 限制
- 角色模块用 `<g transform="translate(x,y)">` 定位

### 实现注意事项

- 所有文字用 `<text>` 元素直接绘制，不依赖 web font
- 箭头使用 SVG `<defs><marker>` 定义，全局复用
- 曲线路径用 `M ... C ...` 贝塞尔曲线，平滑连接各节点
