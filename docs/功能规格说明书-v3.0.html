大屏看板
📌 平台全局数据驾驶舱，给运营和超管用的实时大盘，一眼看清全平台营收、门店、设备、会员的概貌。

📋 规则说明

  • 数据每30秒自动刷新一次，右上角有手动刷新按钮
  • 顶部时间范围（今日/本月/本年）切换后联动刷新所有图表
  • KPI指标的环比变化用↑绿色/↓红色标记，持平箭头灰色
  • 大屏模式下隐藏侧边栏和顶栏，仅展示看板内容

🖱️ 交互说明

  • 登录后默认进入看板，看到5个KPI指标卡（总GMV/今日营收/门店数/会员数/在线设备）
  • 切换顶部时间范围（今日/本月/本年），下方所有图表同步更新
  • 营收趋势折线图：鼠标悬停在数据点显示具体数值，滚轮可以缩放时间轴
  • 门店营收排行柱状图：前3名颜色不同，点击柱状可下钻查看该门店详情
  • 支付渠道分布饼图：显示各渠道占比和实际金额
  • 热门游戏TOP10表格：点击任意一行跳转到游戏详情页
门店列表
📌 管理全平台所有门店信息，支持查看、搜索、编辑门店资料，禁用异常门店。

📋 规则说明

  • 门店有3种状态：营业中、已关闭、待审核
  • 禁用门店后，该门店所有设备停止接单，但已进行的游戏不受影响
  • 门店信息编辑后立即生效，无需审核
  • 导出Excel最多支持10000行，超过时提示缩小筛选范围

🖱️ 交互说明

  • 顶部筛选栏按门店名称、所属代理、状态、地区进行筛选
  • 表格中每行显示门店名称、地址、代理、状态等信息
  • 点击「查看」弹出详情抽屉，展示基本信息 + 关联设备列表 + 近7日营收趋势
  • 点击「编辑」弹出表单修改门店信息
  • 点击「禁用」弹出确认框，确认后门店暂停营业
  • 点击「导出」将当前筛选结果下载为Excel文件
商家列表
📌 管理全平台商家账户，审核商家入驻、查看商家经营数据、管理商家状态。

📋 规则说明

  • 商家状态流转：注册 → 提交资料 → 平台审核（通过/驳回）→ 正常经营
  • 冻结商家后，该商家下所有门店暂停营业
  • 重置密码会发送新密码到商家注册手机号
  • 商家详情中显示该商家的游戏豆库存和采购记录
  • 批量审核时可以选择通过或驳回，驳回需填写原因

🖱️ 交互说明

  • 筛选栏按商家名称、状态、所属代理、注册时间筛选
  • 商家列表展示名称、联系人、手机号、门店数、状态、注册时间
  • 点击「查看详情」进入商家详情页，包含基本信息 + 门店列表 + 订单统计 + 游戏豆库存
  • 点击「冻结」弹出二次确认，确认后商家暂停
  • 点击「重置密码」触发密码重置流程
  • 支持批量勾选商家进行审核或导出
代理商列表
📌 管理全平台代理商，审核入驻、查看分润数据、管理代理级别和保证金。

📋 规则说明

  • 代理商分三级：城市代理/区域代理/省级总代，每级别基础分润率不同
  • 保证金状态：已缴纳/不足/未缴纳，保证金不足时系统自动提醒
  • 代理级别变更后，从下月起按新级别计算分润
  • 级别变更时系统自动评估影响范围（涉及哪些商家、金额变化）

🖱️ 交互说明

  • 按代理商名称/编号、级别、状态、地区筛选
  • 列表展示名称、级别、管理区域、旗下商家数、本月分润预估、保证金状态
  • 点击「查看详情」进入详情页：基本信息 + 旗下商家 + 分润历史 + 提现记录
  • 点击「编辑级别」弹出级别变更面板，显示变更影响评估
  • 保证金管理：查看缴纳记录，手动标记已缴纳
游戏库管理
📌 管理平台上架的所有VR游戏的信息、版本和分发范围。

📋 规则说明

  • 游戏有3种状态：已上架/已下架/待审核
  • 下架游戏后，已分发的门店不受影响（已下载的包体仍可使用），新用户不可选
  • 每个游戏可以有多个版本，但同一时间只有一个版本是活跃版本
  • 游戏包体上传至MinIO/OSS存储，大文件支持断点续传

🖱️ 交互说明

  • 游戏以卡片网格展示，按分类筛选（刺激/恐怖/休闲/亲子）
  • 点击卡片进入游戏详情页，包含基本信息 + 版本列表 + 分发配置
  • 「上架新游戏」填写名称、分类、封面图、视频、描述、默认价格、建议时长
  • 版本管理中上传游戏包体，选择是否设为当前版本
  • 分发配置中选择哪些代理商或门店可以使用该游戏
  • 支持批量上下架游戏
内容分发中心
📌 平台运营将游戏、广告、活动等内容定向分发给指定门店或代理商。

📋 规则说明

  • 分发任务状态：待分发 → 分发中 → 已结束/已取消
  • 定时分发任务到达设定时间自动触发
  • 已分发的游戏会自动推送到目标门店的VR终端上
  • 分发范围可选：全平台、指定代理商（旗下所有门店）、指定门店列表
  • 提前结束分发后，已推送的内容不会撤回，但新用户不可选

🖱️ 交互说明

  • Tab切换待分发/已分发/分发计划
  • 创建分发任务：选择内容类型（游戏/广告/活动）→ 选择目标范围 → 设置生效时间 → 提交
  • 待分发列表：查看已创建但未生效的任务，可编辑或取消
  • 已分发列表：查看正在生效的任务，可查看详情或提前结束
  • 分发计划列表：管理定时分发的任务，可暂停/恢复
营收总览
📌 平台级财务仪表盘，展示营收趋势、收入构成和各门店营收排行。

📋 规则说明

  • 营收分为两条线：游戏豆销售额 和 硬件销售佣金（硬件不通过平台系统）
  • 数据可导出Excel，上限10000行
  • 同比环比按自然月计算

🖱️ 交互说明

  • 选择时间范围（今日/本周/本月/本年/自定义）
  • 查看KPI卡片：总营收、游戏豆销售额、硬件销售额、平台净利润
  • 营收趋势折线图按日/周/月聚合展示
  • 收入构成饼图展示各收入来源占比
  • 门店营收排行TOP10柱状图
  • 点击导出按钮下载当前筛选数据
游戏豆销售管理
📌 查看全平台游戏豆销售情况：谁买了、买了多少、趋势如何、哪些商家库存不足。

📋 规则说明

  • 销售明细记录每一笔采购订单，包含商家和金额
  • 库存预警阈值由商家自行设置，本页面只展示汇总预警
  • 采购记录不可删除或修改，只能查看

🖱️ 交互说明

  • 顶部统计卡片展示本月销售总额、总销量、采购商家数
  • Tab切换销售明细/采购排行/采购趋势
  • 销售明细表格按日期/商家筛选
  • 采购排行柱状图展示TOP20商家
  • 采购趋势折线图按月展示
  • 低库存预警列表自动显示低于阈值的商家
代理商结算管理
📌 平台方查看和管理所有代理商的月度分润数据，审核结算单、生成打款任务。

📋 规则说明

  • 每月1日系统自动汇总上月各代理商的分润数据
  • 分润 = 辖区商家游戏豆采购总额 × 基础比例 × 阶梯系数（全量阶梯，非分段累进）
  • 结算单状态流转：待结算 → 待确认 → 已确认 → 已打款 → 已完成
  • 分润金额 < ¥100 自动累积到下月
  • 有争议的结算单标橙色，代理商申诉后结算单冻结，需平台财务复核后手动处理
  • 硬件销售分润按线下合同执行，不在此页面处理

🖱️ 交互说明

  • 默认显示上月的结算数据，可切换月份和代理商级别筛选
  • 表格展示每个代理商的硬件分润、游戏豆分润、总分润和状态
  • 点击「查看明细」弹出该代理商的分润计算过程（含阶梯系数应用方式）
  • 点击「生成结算单」系统按阶梯策略计算并生成待确认结算单
  • 批量勾选多个代理商 → 批量生成结算单 → 推送到打款队列
  • 流水为0的代理商灰显，有争议的标橙色，分润不足¥100的提示累积
阶梯分润策略配置
📌 超管配置全平台的分润阶梯策略，决定不同代理级别在不同采购额区间拿到的分润比例。

📋 规则说明

  • 分三级代理配置（城市/区域/省级），每级有独立的基础比例和阶梯区间
  • 计算方式：全额按档位计算（落在哪个区间就全员按该系数），不是分段累进
  • 策略有版本管理，新发布策略默认下月起生效
  • 历史版本保留，支持一键回滚到任一历史版本
  • 发布前必须进行影响评估，显示受影响的代理商数量和分润金额变化范围

🖱️ 交互说明

  • Tab切换城市代理/区域代理/省级总代
  • 当前策略卡片显示生效版本的摘要
  • 编辑时设置基础比例和4档阶梯区间的系数
  • 输入假设采购额，实时预览计算结果
  • 保存时自动弹出影响评估弹窗
  • 确认后发布新版策略，设定生效时间
打款管理
📌 结算单生成后的监控和异常处理中心。正常单自动提交拉卡拉分账，异常单需人工处理。

📋 规则说明

  • 自动打款条件：结算金额≥¥100、代理账户信息完整、无争议/申诉
  • 打款失败自动重试3次，3次均失败后标记为失败需人工介入
  • 失败单可转为线下打款（需上传银行回单凭证）
  • 打款账户修改有10分钟冷却期 + 短信验证码二次确认

🖱️ 交互说明

  • 状态Tab切换：全部/待打款/打款中/已打款/打款失败
  • 待打款Tab展示已审核通过的结算单，系统自动处理
  • 打款中Tab展示正在处理中的单子和进度
  • 打款失败Tab展示失败原因，支持「重试」或「转为线下打款」
  • 点击结算单号弹出完整打款详情（结算周期+分润明细+打款凭证）
  • 统计卡片展示本月打款总金额、成功/失败笔数、平均处理时长
对账中心
📌 平台账与支付渠道账、商家账、代理商账的四方自动比对和差异处理。

📋 规则说明

  • 四方数据自动拉取：平台订单数据、微信/支付宝渠道账单、商家结算记录、代理商分润记录
  • 不一致的金额红色高亮标记
  • 人工调账后需记录操作原因，调账结果不可撤销
  • 对账报告支持Excel和PDF两种导出格式

🖱️ 交互说明

  • 选择对账周期（日/月）和日期
  • 点击「开始对账」自动拉取各方数据比对
  • 比对结果以矩阵表格展示，差异项红色高亮
  • 点击差异行查看明细（平台 vs 渠道 vs 商家 vs 代理）
  • 确认差异归属后，选择「以平台为准」「以渠道为准」或「手动修正」
  • 导出对账报告
设备总览
📌 查看全平台所有VR设备的状态分布、在线率和使用率。

📋 规则说明

  • 设备状态：在线（绿）/ 使用中（蓝）/ 离线（灰）/ 故障（红）/ 维修中（黄）
  • 设备离线超过15分钟自动标记为离线
  • 设备的心跳数据每60秒上报一次

🖱️ 交互说明

  • 统计卡片展示总设备数、在线数、使用中、离线、故障数量
  • 设备状态分布饼图展示各状态占比
  • 在线率趋势折线图展示近7天变化
  • 设备列表按编号/类型/所属门店筛选，展示状态和最后心跳时间
  • 点击设备编号进入设备详情（参数+运行日志+心跳记录）
消息推送
📌 平台运营向代理商和商家发送系统公告和推送消息。

📋 规则说明

  • 公告接收对象可选：全部/指定代理商/指定门店
  • 已发布的公告支持撤回，撤回后接收方看不到
  • 已读统计只统计用户主动点击查看的行为
  • 草稿箱中的内容不通知任何人

🖱️ 交互说明

  • Tab切换公告管理和推送记录
  • 创建公告：输入标题 + 编辑内容（markdown编辑器） + 选择接收对象 + 发布
  • 创建推送：选择推送类型（系统通知/活动通知/告警）+ 内容 + 目标
  • 草稿箱可暂存未发布的公告
  • 已发布列表展示标题、发送时间、接收范围、已读量
系统设置（RBAC）
📌 平台的权限管理和系统配置中心，管理角色、权限、操作日志和系统参数。

📋 规则说明

  • 8个系统角色：超管/运营/财务/内容/代理商/店长/收银员/员工
  • 权限粒度为页面级 + 操作级（查看/创建/编辑/删除）
  • 操作日志保留至少180天
  • 角色权限变更后立即生效

🖱️ 交互说明

  • Tab切换角色管理/权限配置/操作日志/系统参数
  • 角色管理展示角色列表，点击进入权限编辑
  • 权限配置以权限树形式勾选，按模块/页面/操作三层粒度
  • 操作日志按时间/操作人/IP搜索过滤
  • 系统参数以key-value形式配置

工作台概况
📌 商家登录后第一眼看到的页面，展示今日核心运营数据和快捷操作入口。

📋 规则说明

  • 数据每30秒自动刷新
  • 设备使用率超过90%时告警提示
  • 今日营收数据截止到当前时刻，非全天数据

🖱️ 交互说明

  • 查看KPI卡片：今日营收、今日客流、设备使用率、新增会员、退款笔数
  • 查看设备状态卡片，了解各设备实时状态
  • 查看今日营收趋势简易折线图（按小时）

会员管理
📌 管理门店会员，包括会员信息维护、储值消费、积分和等级管理。

📋 规则说明

  • 会员等级：普通/银卡/金卡/钻石/黑金，各级别折扣和权益不同
  • 会员储值支持套餐充值和自定义金额充值
  • 积分获取规则：消费金额 × 等级倍数（由商家在后台配置）
  • 手动调整积分需要填写原因，操作计入审计日志
  • 删除会员为逻辑删除，数据保留但不再展示
  • 会员手机号唯一，重复注册时提示已有账号

🖱️ 交互说明

  • 搜索会员：按手机号/昵称搜索，按等级/标签筛选
  • 添加会员：填写姓名/手机号/生日/性别/备注
  • 点击会员行进入详情页：查看基本信息 + 储值余额 + 积分 + 消费记录 + 充值记录
  • 会员充值：选择充值套餐或自定义金额 → 选择支付方式 → 确认
  • 会员等级配置：设置各等级对应的折扣和权益
  • 积分管理：查看积分变动日志，手动调整积分
商品（游戏项目）管理
📌 管理门店销售的游戏项目，包括定价、消耗游戏豆数和库存上下架。

📋 规则说明

  • 商品类型：按次收费 / 按时长收费 / 套餐包
  • 消耗游戏豆数表示C端用户玩一次该游戏商家需消耗多少个游戏豆
  • 库存不足时自动标红预警
  • 下架商品后，C端用户在PC终端上不可见，但已有未使用的订单仍有效
  • 批量调价支持按比例上调/下调或设置固定值

  • 展示头号空间Logo和动态待机画面
  • 设备在线时心跳上报（每60秒一次）
  • 设备离线超过15分钟自动标记离线

🖱️ 交互说明

  • 待机状态下自动播放品牌动画 → 收到launch_game指令后自动拉起游戏
游戏加载
📌 收到PC终端下发的启动指令后，自动加载并启动游戏。

📋 规则说明

  • 通过MQTT接收launch_game指令，指令包含游戏包ID和Session Token
  • 加载进度实时上报到后端，PC终端同步显示加载进度
  • 加载超时（超过120秒）自动标记失败，退还用户支付

🖱️ 交互说明

  • PC终端支付成功 → 后端下发launch_game → VR头显自动加载游戏 → 加载进度同步到PC终端
游戏游玩
📌 游戏运行中的核心体验阶段，全程零UI沉浸设计。

📋 规则说明

  • 游戏运行期间持续上报心跳和计费数据
  • 计费按实际游玩时长计算（精确到秒）
  • 游戏结束时自动上报Session结算数据
  • 设备异常退出时记录错误码并上报

🖱️ 交互说明

  • 游戏全程仅显示Logo+文字提示，不展示任何UI界面
  • 结束时跳转到结算画面，显示消费明细
摘盔暂停
📌 用户摘下头盔时自动暂停游戏，防止操作中断和浪费时长。

📋 规则说明

  • 检测到头显脱离面部后自动暂停游戏并冻结计时
  • 暂停超过3分钟自动结束游戏并结算（按实际游玩时长收费）
  • 重新佩戴后自动恢复游戏

🖱️ 交互说明

  • 摘盔 → 游戏画面冻结 + 暂停计时 → 3分钟内佩戴继续游戏 → 超时自动结束返回结算
OTA升级
📌 远程固件更新和游戏包体推送，无需人工操作。

📋 规则说明

  • OTA升级包由平台后台统一管理分发
  • 升级期间设备不可用，升级完成后自动重启
  • 升级失败自动回滚到上一版本
  • 支持固件升级和游戏内容远程下装

🖱️ 交互说明

  • 平台推送升级包 → 待机状态下自动下载 → 下载完成后提示重启安装 → 升级完成自动恢复待机
