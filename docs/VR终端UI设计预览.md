# VR 头显终端 · 沉浸式界面设计

> 头号空间 v1.3 — 纯沉浸式体验，零干扰 UI

## 设计原则

VR 终端**不做任何可有可无的 UI**。

用户戴上头盔的唯一目的就是沉浸式玩游戏。

所有操作（选游戏、支付、监控、结算）全部由 **PC 终端** 完成。VR 终端只负责三个必要条件：

1. 加载过渡
2. 纯游戏
3. 结束提示

## 状态流转

```
待机 Idle → 加载 Loading → 纯游戏 GameRunning → 结束/异常 Ended/Error → 回到待机 Idle
```

## 五种 VR 界面状态

### 1. Idle — 待机

- **触发**：设备空闲，等待 PC 终端分配
- **视觉**：Logo 常亮浮空
- **行为**：30 秒后自动淡出至纯黑（省电）
- **交互**：用户无任何可交互元素

### 2. Loading — 加载中

- **触发**：PC 终端下发游戏启动指令
- **视觉**：Logo + 旋转加载环 + "正在启动游戏..."
- **行为**：最多显示 3 秒，游戏进程启动后自动切换
- **交互**：用户无需任何操作

### 3. GameRunning — 游戏中

- **触发**：游戏进程启动成功
- **视觉**：纯游戏画面，零 UI 叠加
- **禁止项**：
  - 无倒计时 HUD
  - 无系统菜单入口
  - 无进度条叠加
- **后台**：SDK 做心跳 + 计时 + 摘盔检测，用户零感知

### 4. GameEnded — 体验结束

- **触发**：时间到 / PC 终端远程结束 / 摘盔超时
- **视觉**：一行白色文字 "体验已结束 请取下头盔"
- **行为**：3 秒后自动返回待机
- **注意**：不展示任何消费金额/时长/余额，结算细节请到 PC 终端查看

### 5. Error — 系统异常

- **触发**：游戏崩溃 / SDK 异常 / 设备过热
- **视觉**：⚠️ 图标 + "系统异常 请联系店员"
- **行为**：文字常驻不消失，店员在 PC 终端查看日志并重启
