# 设备管理逻辑说明

## 设备类型

| 设备 | 角色 | 需要 Token? |
|------|------|------------|
| **主机** (PC终端) | 运行点播系统，连接服务器接收指令 | ✅ **需要** |
| **头显** (VR眼镜) | 连接主机进行游戏体验 | ❌ 不需要 |

---

## 设备使用流程

### 主机

```
平台录入主机（填写编号、名称、MAC地址等）
    ↓
平台分配主机到商家 / 店铺
    │
    ├──→ 分配成功 → 自动生成 Token（身份凭证）
    │                ↓
    │              主机启动 → 用 Token 连接平台服务器
    │                ↓
    │              服务器验证 Token → 确认主机身份
    │                ↓
    │              主机正常工作：
    │                ├─ 接收游戏分发
    │                ├─ 接收远程控制指令
    │                ├─ 接收 OTA 升级
    │                ├─ 上报心跳和运行状态
    │                └─ 转发指令到头显
    │
    └──→ 取消分配 → Token 自动吊销 → 主机下次上线被拒绝
```

### 头显

```
平台录入头显（填写名称、型号、SN码等）
    ↓
平台分配头显到商家 / 店铺
    ↓
商家门店在系统后台 → 绑定头显到指定主机
    ↓
头显通过 WiFi Direct 直连主机
    ↓
主机转发游戏画面和指令到头显
```

---

## Token 说明

### Token 是什么

Token 是**主机的身份凭证**，相当于主机的"密码"。每台主机分配商家时自动生成唯一 Token。

### Token 的作用

| 功能 | 说明 |
|------|------|
| **身份认证** | 主机连接服务器时用 Token 证明身份 |
| **安全隔离** | Token 唯一，防止未授权设备接入 |
| **权限控制** | 服务器根据 Token 识别设备所属商家/店铺 |

### Token 生命周期

```
分配主机 → 自动生成 Token（active）
    ↓
主机用 Token 连接服务器 → 正常工作
    ↓
取消分配 → Token 自动吊销（none）
    ↓
再次分配 → 重新生成新 Token
```

### 操作入口

Token 管理已合并到**主机管理**表格中（运营后台 → 设备配置管理）：

| 操作 | 说明 |
|------|------|
| **查看 Token** | 主机分配后自动生成，表格中显示脱敏 Token |
| **复制 Token** | 点击复制按钮，用于线下配置主机 |
| **吊销 Token** | 手动吊销，主机下次上线被拒绝，需重新生成 |
| **生成 Token** | Token 吊销后重新生成 |

## 与商家端的关系

```
平台录入 + 分配
    ↓
商家店铺：
  ├── 查看已分配的主机和头显
  ├── 绑定头显到主机
  ├── 查看主机在线状态、电量等
  └── 接收平台分发的游戏到主机
```

商家端只能**使用**已分配的设备（查看、绑定、监控），不能录入或分配。
