# 头号空间 · 资金流通全景图

> C端消费 → 商家运营 → 平台枢纽 → CP结算 → 代理分润，全链路资金流转

## 六大资金流

### ① C端消费流 — 消费付款

- **流向**：C端用户 → 商家/门店
- **方式**：微信 / 支付宝 / 现金
- **规则**：100% 归商家

### ② 游戏豆采购流 — 采购游戏豆

- **流向**：商家/门店 → 平台运营
- **定价**：¥1/豆
- **方式**：微信 / 支付宝 / 对公转账

### ③ 代理分润流 — 代理分润

- **流向**：平台运营 → 代理商
- **规则**：按辖区采购额阶梯分润 3%~12%
- **结算**：次月 15 日拉卡拉打款

### ④ 商家提现流 — 商家提现

- **流向**：商家/门店 → 银行账户
- **规则**：扣手续费
- **结算**：拉卡拉 T+1 打款

### ⑤ 退款回流

- **规则**：
  - 未结算：原路退回
  - 已结算：线下退回
- **流向**：商家/门店 → C端用户

### ⑥ CP分账流 — CP 分账

- **流向**：平台运营 → CP 供应商
- **规则**：有效体验 × 成本价分成
- **结算**：T+1 拉卡拉周期打款

## 角色定义

### C 端用户

- 身份：游客 / 会员
- 行为：到店消费 VR 体验、充值会员余额、按次付费

### 商家 / 门店

- 身份：收银 · 运营
- 收入：消费收银 100% 归商家
- 支出：消耗游戏豆启动 VR 游戏

### 平台运营

- 身份：超管 · 财务中心
- 职责：游戏豆统一定价 ¥1/豆、对账、分账、打款、CP 结算

### 代理商

- 层级：城市 / 区域 / 省级
- 收入：按辖区采购额阶梯分润
- 结算：次月 15 日拉卡拉打款

### CP 供应商

- 身份：游戏内容提供方
- 收入：有效体验 × 成本价分成
- 结算：拉卡拉周期打款

### 银行账户

- 来源：拉卡拉 T+1 打款
- 用途：商家提现收款
