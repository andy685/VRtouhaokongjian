## 一、标记 / 取消标记优化

### 变更前
- 点击「标记异常」→ 按钮变为灰色禁用的「已标记」
- 无法撤销，点错只能去对账中心删除

### 变更后
- 未标记 → `⚠ 标记异常`（橙色，可点击）
- 已标记 → `⚠ 取消标记`（橙色，可点击，点击即从异常列表移除）

### 涉及文件

| 文件 | 改动 |
|------|------|
| `composables/useExceptionOrders.ts` | 新增 `unmarkOrder(orderNo)` 方法 |
| `views/shop/CashierOrder.vue` | 按钮改为可取消 |

## 二、驳回流程补全

### 变更前
- 平台驳回后，商家端状态列掉到兜底「待处理」，看不到驳回原因
- 没有「重新提交」入口

### 变更后
**商家端（shop/Reconciliation.vue）**
- 状态列新增 `rejected` → 「已驳回」（红色标签）
- 弹窗展示驳回原因（红色高亮）+ ⚠ 提示修正后重提
- 处理方式二选一：
  - 修正后重新提交平台 → `status = escalated`
  - 自行解决 → `status = resolved`
- 按钮显示「重新处理」

**平台端（platform/Reconciliation.vue）**
- 状态列 `escalated` → 「商家重提」（蓝色标签）
- 操作按钮统一：非 `resolved` 均显示「处理」

### 状态流转

```
pending ──→ escalated ──→ rejected ──→ escalated ──→ resolved
(待处理)    (平台处理中)   (已驳回)     (重新提交)     (已处理)
                            │
                            └──→ resolved（商家自行解决）
```

## 四、收银系统（cashier-ui）评估结论

- cashier-ui 是完全独立的项目（Element Plus），不共享 admin-dashboard 代码
- 无异常标记功能，**评估结论：不需要补充**
- 理由：收银台职责是快速收银，异常标记属于管理后台的对账环节
- 一人店场景使用 admin-dashboard 商家端即可覆盖
