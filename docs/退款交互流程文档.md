# 退款交互流程文档

> 备份日期：2026-07-10（v1.1 - 补充线下退款凭证必填规则 + 整合参考图版式）
> 涉及项目：`cashier-ui`（收银系统） + `admin-dashboard`（管理后台）
> 参考设计稿：`docs/images/线下退款参考图.png`
>
> **核心规则：**
> 1. 因技术方案限制，**仅支持整单退款**，不支持部分退款
> 2. 线下退款（已结算订单）**必须上传线下退款凭证**（转账截图等）

---

## 一、退款入口与触发条件

### 适用页面

| 项目 | 页面 | 路由 |
|------|------|------|
| cashier-ui | 收银订单 | `/order` |
| cashier-ui | 点播系统订单 | `/order/vod` |
| admin-dashboard | 收银订单 | shop/CashierOrder |
| admin-dashboard | 点播系统订单 | shop/OnDemandOrder |
| admin-dashboard | 订单查询 | shop/OrderSearch |

### 退款按钮显示条件

```
订单状态 === '完成'  AND  showRefund === true
```

- **收银订单** 和 **点播系统订单**：`showRefund = true`
- **手动扣款订单**：`showRefund = false`（不显示退款按钮）

按钮文案根据**结算状态**区分：
- 已结算（settled）：显示 **「线下退款」**（橙色按钮）
- 未结算（pending）：显示 **「申请退款」**（红色按钮）

## 二、退款交互流程详解（cashier-ui）

### 步骤 1：点击「详情」→ 打开订单详情弹窗

```
[订单列表页]
   │
   └── 点击表格中某行的「详情」按钮
        │
        └── 打开订单详情弹窗 (ors-detail-modal)
             ├── 订单基本信息（订单号、金额、状态、支付方式）
             ├── 已退金额、退款原因（如有）
             └── 底部操作按钮区域
                  ├── 状态=完成 + 已结算  → [线下退款]
                  ├── 状态=完成 + 未结算  → [申请退款]
                  └── 状态≠完成           → 无退款按钮
```

### 步骤 2：点击退款按钮 → 打开退款弹窗

```
[点击「申请退款」或「线下退款」]
   │
   └── 打开退款弹窗 (ors-refund-card)
```

**弹窗标题**：
- 未结算：`申请退款`
- 已结算：`线下退款`

### 步骤 3：退款弹窗内容（按参考图版式）

> 参考图位置：`docs/images/线下退款参考图.png`

#### 3a. 顶部提示条

| 结算状态 | 样式 | 文案 |
|----------|------|------|
| 未结算 | 蓝色提示 | 「原路退回，退款总金额 ¥xxx.xx 元」 |
| 已结算 | 黄色警告 | 「该订单已结算，款项已到商家账户。请商家线下退款给用户后，上传凭证并确认。」 |

> ✅ 提示文案中需显示**退款总金额**（按整单计算）。

#### 3b. 订单信息区（4 字段网格，参考图版式）

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 订单号 | 202607091544202909140159 | 完整订单号 |
| 退款内容 | 海底两万里 | 商品名（蓝色文字样式） |
| 退款金额 | ¥100.00 | 红色文字突出显示 |
| 退款方式 | 原路退款 / 线下退款 | 标题为「退款方式」 |

#### 3c. 退款路径区（仅未结算·原路退回 时显示）

| 字段 | 说明 |
|------|------|
| 支付方式 | 如「会员卡余额」蓝色文字 |
| 支付渠道 | 如「内部账户」 |
| 退款金额 | 各项拆分后的金额 |
| 说明 | 如「原路退回至会员卡余额」 |

> 当前仅支持**单一支付方式**的原路退回；组合支付按比例拆分（具体拆分规则后续完善）。

#### 3d. 退款原因（必填）

下拉选择框，选项：

| 序号 | 退款原因 |
|------|----------|
| 1 | 客户不想要了 |
| 2 | 游戏体验异常 |
| 3 | 操作错误（买错项目） |
| 4 | 设备故障 |
| 5 | 重复付款 |
| 6 | 其他 |

> admin-dashboard 中若选择「其他」，会额外显示文本框供填入自定义原因。

#### 3e. 底部汇总区（参考图版式）

```
退款方式：原路退款
本次退款总额：¥100.00
会员卡余额：¥100.00
```

| 字段 | 未结算 | 已结算 |
|------|--------|--------|
| 退款方式 | 原路退款 | 线下退款 |
| 本次退款总额 | 现单价 × 整单数（红色） | 同左 |
| 渠道补充说明 | 「会员卡余额：¥xxx.xx」 | 「现金/转账：¥xxx.xx」 |

#### 3f. 线下退款凭证上传（⚠️ 必填，仅已结算订单显示）

**【强约束】线下退款必须上传凭证**

| 项目 | 说明 |
|------|------|
| 是否必填 | ✅ **必填**，未上传时确认按钮禁用 |
| 支持格式 | 图片（jpg/png/jpeg），单文件 ≤ 5MB |
| 上传数量 | 至少 1 张，最多 3 张 |
| 字段名称 | `refundVoucher` |
| 文件命名建议 | `refund_{orderNo}_{timestamp}.{ext}` |
| UI 组件 | `n-upload`（admin-dashboard） / 自定义上传区（cashier-ui） |

**校验逻辑**：
```javascript
const canConfirmRefund = computed(() => {
  if (refundQty.value <= 0) return false
  if (!refundReason.value) return false
  // 已结算订单必须上传凭证
  if (isSettled.value && refundVoucherList.value.length === 0) return false
  return true
})
```

#### 3g. 确认按钮

| 结算状态 | 按钮文案 | 启用条件 |
|----------|----------|----------|
| 未结算 | **确认退款** | refundQty > 0 && refundReason 非空 |
| 已结算 | **确认已线下退款** | refundQty > 0 && refundReason 非空 **&& 凭证已上传** |

### 步骤 4：确认退款 → 后端处理

#### 未结算订单（拉卡拉原路退回）

```
确认退款
  │
  ├── 订单状态 完成 → 退款中
  ├── 退款方式 = '原路退款'（即拉卡拉原路退回）
  ├── 退款路径明细 = 按支付方式拆分
  ├── 即时提示：「退款请求已提交，等待拉卡拉处理...」
  │
  └── (2秒后异步回调)
       ├── 订单状态 退款中 → 已退款
       ├── 生成拉卡拉退款单号 RF + 时间戳
       ├── 更新 refundAmount、refundedQty
       └── 提示：「拉卡拉退款成功，退款单号 RFxxx」
```

#### 已结算订单（线下退款）

```
确认线下退款
  │
  ├── 订单状态 完成 → 已退款
  ├── 退款方式 = '线下退款'
  ├── 上传凭证文件 refundVoucher（必填）
  │   ├── 保存到对象存储（如 COS/OSS）
  │   └── 记录 voucherUrl[] 数组到订单
  ├── 更新 refundAmount、refundedQty
  └── 提示：「已标记线下退款，退款金额 ¥xxx」
```

#### 后端提交字段汇总

```typescript
interface RefundRequest {
  orderId: string                   // 订单ID
  refundType: 'online' | 'offline'  // online=原路退回, offline=线下退款
  refundAmount: number              // 退款金额（整单）
  refundReason: string              // 退款原因
  refundPathList?: {                // 仅 online 需要，原路退回路径
    payMethod: string
    payChannel: string
    amount: number
  }[]
  voucherUrls?: string[]            // 仅 offline 需要，线下凭证 URL 数组
}
```

### 步骤 5：关闭弹窗 → 列表更新

```
关闭退款弹窗 + 关闭详情弹窗
  │
  └── 订单列表中：
       ├── 状态标签变为红色「已退款」(statusTone = 'refund')
       ├── 未结算订单结算状态变为灰色「无需结算」
       └── 退款金额和退款原因字段更新
```

## 三、admin-dashboard 中的退款流程

admin-dashboard 中的 `CashierOrder.vue` 和 `OnDemandOrder.vue` 实现了与 cashier-ui 类似的退款弹窗，使用 **Naive UI** 组件体系。

关键差异：
- 使用 `n-modal` 弹窗、`n-select` 下拉、`n-upload` 上传等 Naive UI 组件
- 已结算退款需要上传凭证文件（`n-upload`）
- 未结算退款先变为「退款中」，2秒后模拟拉卡拉回调，变为「已退款」
- 选择退款原因「其他」时，出现额外文本输入框

## 七、当前技术限制与业务规则说明

### 技术限制：仅支持整单退款

> ⚠️ **重要**：当前因拉卡拉分账接口技术方案限制，**仅支持整单退款**。
>
> 具体影响：
> - 虽然 UI 上保留了数量选择器，但实际只能退整单
> - 组合支付的订单也仅能整单退，不支持按支付方式部分退款
> - 如需支持部分退款，需等拉卡拉接口升级或更换支付通道

### 业务规则：线下退款必须上传凭证

> 📌 **业务约束**：无论整单/部分退款场景，选择 **「线下退款」**（即已结算订单退款）时，**必须** 上传线下退款凭证（如转账截图、现金收据照片等）。
>
> 实施规则：
> - 凭证必填，否则确认按钮禁用
> - 支持图片格式（jpg/png/jpeg），单文件 ≤ 5MB
> - 至少 1 张，最多 3 张
> - 凭证作为订单字段 `refundVoucherList[]` 持久化保存
> - 凭证 URL 应存储到对象存储（COS/OSS），不直接保存 base64
>
> 业务目的：
> - 财务对账留痕，避免商家线下退款的扯皮风险
> - 平台端对账审核时有据可查
> - 发生纠纷时可作为证据追溯
