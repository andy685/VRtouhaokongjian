## 🎯 一、项目定位

**做什么**: 一套统一的VR体验店运营管理后台

### 2.4 具体截图对比证据

#### 对比1: 登录体验
```
幻影星空:
  · 显示"当前为游客模式, 登录后可使用全部功能"
  · 用户感知: 功能受限，需要额外操作
  
我们改进 (见 huanying_login_page.png):
  · 渐变蓝紫背景，科技感强
  · 支持手机验证码 + 微信扫码双方式

```

#### 对比2: Token机制
```
幻影星空:
  · 有Token但操作路径不清晰
  · 需手动复制粘贴，易出错
  
我们改进 (见设计方案829-863行):
  · 店铺设置 → 收银设置 一键复制
  · 可视化使用步骤指引
  · 支持重新生成 + 操作审计日志
  · 多端同步(PC收银+小程序)
```

#### 对比3: 工作台设计
```
幻影星空 (overview_full.png):
  · 基础数据卡片堆叠
  · 设备状态仅文字显示
  
我们改进 (见设计方案544-597行):
  · 今日概况 + 待办事项 双栏布局
  · 设备卡片网格: 运行/空闲/故障/离线 四态
  · 每台设备显示: 当前游戏 + 剩余时间
  · 营收趋势图 + 热门游戏TOP5 并列
```

## 👥 四、权限体系（8个角色）

### 层级结构

```
L1 平台层:  超级管理员 | 平台运营 | 平台财务 | 内容审核
L2 区域层:  代理商管理员
L3 门店层:  店长 | 收银员 | 员工
```

### 核心设计原则
- **超管**: 全局权限 `*`
- **数据隔离**: 代理商只能看代理门店，店长只能看本店
- **功能限制**: 收银员只能开单，不能看报表

## 🔴 五、总运营后台（5大模块）

### 模块概览

| # | 模块 | 核心功能 | 优先级 |
|---|------|----------|--------|
| 1 | **📊 数据中心** | 全局大屏 + 自定义报表 | P0 |
| 2 | **🏪 门店体系** | 门店CRUD + 审核 + 等级套餐 | P0 |
| 3 | **🎮 内容中心** | 游戏库 + 分类标签 + 分发推送 | P1 |
| 4 | **💰 平台财务** | 结算规则(15%抽成) + 打款 | P1 |
| 5 | **⚙️ 运维中心** | 版本发布 + 告警系统 | P2 |

### 关键业务规则

**门店等级套餐**:
- ⭐ 基础版 ¥299/月 (≤10台设备)
- ⭐⭐ 专业版 ¥599/月 (≤30台) 
- ⭐⭐⭐ 旗舰版 ¥999/月 (≤100台)
- 🏆 定制版 面议

**结算规则**:
- 平台抽成: 15%（基础）
- 返利政策: 月营收≥50K减2%, ≥100K减3%, ≥200K减5%
- 结算周期: 月结，次月15日打款

## 🟢 六、店铺运营后台（6大模块）

### 模块概览

| # | 模块 | 核心功能 | 使用频率 |
|---|------|----------|----------|
| 1 | **📈 工作台** | 今日概况 + 设备实时状态 + 待办 | 每日必看 |
| 2 | **👥 会员管理** | 5级会员体系 + 储值 + 标签画像 | 高频 |
| 3 | **🎮 设备管理** | 设备列表 + 远程控制(重启/结束/喊话) | 高频 |
| 4 | **💵 营业报表** | 收入趋势 + 支付方式占比 + 时段分布 | 每日/周 |
| 5 | **🎟️ 营销工具** | 5种优惠券类型 + 活动配置 | 按需 |
| 6 | **⚙️ 店铺设置** | **Token配置**(重要!) + 收银参数 | 低频 |

### 🔑 Token机制（关键！）

```
用途: 连接PC收银系统 + 小程序
位置: 店铺设置 → 收银设置
操作: 
  1. 复制Token
  2. PC收银系统 → 设置 → 粘贴
  3. 或小程序后台填入
安全: 支持重新生成 + 审计日志
```
