# 统一交易模块 PRD 更新参考

## 文档目的

本文不替代接口文档。产品 PRD 建议重点补充：业务入口、状态口径、支付/退款/结算规则、异常处理、后台配置项和拉卡拉环境参数。

## 当前覆盖范围

统一交易模块第一阶段覆盖以下业务：

- 收银系统会员充值下单、支付、自动履约。
- 收银系统消费下单、组合支付、退款、补偿履约。
- 终端/PC 点播消费下单、拉卡拉支付。
- 商家后台游戏豆充值下单、支付、自动入账。
- 运营后台交易统计，包括会员充值、客户消费、游戏销售。
- 运营后台商家结算、代理商分润结算、结算单打款。
- 拉卡拉支付、退款、余额分账、账管家提现回调。

第一阶段预留但未完整产品化的能力：

- 优惠券：消费订单 DTO 预留 `couponId`，充值订单不支持优惠券；当前未实现领券、核销、优惠分摊。
- 订单取消、超时关单、支付超时关闭：枚举已预留，当前未形成完整接口和定时任务。
- 线下退款人工处理闭环：已生成待处理退款单，但后台审批、打款凭证、人工完成接口需要产品补充。
- 拉卡拉对账、补单、查询重试：网关接口已预留查询能力，当前产品流程需补充对账任务和人工重试入口。

## 下单参数口径

### 会员充值

必填：

- `storeId`：门店 ID。
- `memberId`：会员 ID。
- `memberCardId`：会员卡 ID。
- `items`：充值套餐明细。

明细字段：

- `itemId`：充值套餐 ID。
- `itemName`：套餐名称快照。
- `quantity`：购买数量，必须大于 0。
- `unitPrice`：单价，必须大于 0。
- `balanceArrivalAmount`：余额到账金额。
- `gameCoinArrivalAmount`：游戏币到账数量。

产品规则：

- 门店充值不允许使用优惠券。
- 支付成功后自动履约：余额和游戏币写入会员卡账户。
- 充值到账金额可能大于实付金额，超出部分按赠送金额处理。

### 收银消费

必填：

- `storeId`：门店 ID。
- `memberCustomer`：是否会员消费。
- `items`：消费项目明细。

按场景填写：

- 会员消费时传 `memberId`。
- 使用会员卡余额或游戏币支付时必须传 `memberCardId`。
- 散客消费可不传会员信息。

产品规则：

- 第一阶段优惠券仅预留，未核销。
- 支付成功后订单自动进入履约流程；当前消费订单履约主要推进交易状态，游戏启动仍由终端业务控制。

### 终端/PC 消费

必填：

- `storeId`：门店 ID。
- `items`：消费项目明细。

可选：

- `terminalDeviceId`：终端设备 ID。
- `memberId`、`memberCardId`：会员消费时传。

产品规则：

- 终端通常使用拉卡拉动态码或聚合收银台。
- PRD 需要补充终端支付状态轮询、支付超时、订单取消和游戏启动条件。

### 商家游戏豆充值

必填：

- `items`：游戏豆充值明细。

明细字段：

- `itemId`：游戏豆套餐或业务项目 ID。
- `itemName`：项目名称快照。
- `quantity`、`unitPrice`：用于计算订单金额。
- `gameBeanAmount`：本次购买游戏豆数量。
- `gameBeanTargetType`：`1` 商家总账户，`2` 门店账户。
- `gameBeanTargetStoreId`：目标为门店账户时必填。

产品规则：

- 支付成功后自动履约，调用游戏豆账户入账。
- 充值到门店账户时，PRD 需要明确门店是否必须属于当前商家。

## 支付规则

支付提交参数：

- `orderNo`：平台订单号。
- `payments`：支付组成项列表。

支付组成项：

- `paymentMethod`：支付方式。
- `paymentChannel`：支付通道。
- `amount`：支付金额，必须大于 0。
- `authCode`：拉卡拉被扫支付必填，即扫码枪扫到的客户付款码。

支付方式：

| 编码 | 方式 |
| --- | --- |
| 1 | 现金 |
| 2 | 支付宝 |
| 3 | 微信 |
| 4 | 会员卡余额 |
| 5 | 会员卡游戏币 |
| 6 | 优惠券预留 |

支付通道：

| 编码 | 通道 |
| --- | --- |
| 1 | 现金 |
| 2 | 内部账户 |
| 3 | 拉卡拉动态码 |
| 4 | 拉卡拉被扫 |
| 5 | 拉卡拉聚合收银台 |

支付组合约束：

- 本次提交的支付金额合计必须等于订单待支付金额；第一阶段不支持长期部分挂账。
- 现金通道只能使用现金支付方式。
- 内部账户通道只能使用会员卡余额或会员卡游戏币。
- 拉卡拉通道只能使用支付宝或微信。
- 拉卡拉被扫支付必须提供 `authCode`。
- 拉卡拉支付统一进入平台拉卡拉钱包，商家和代理商不直接收款。

支付结果：

- 现金、会员卡余额、会员卡游戏币属于内部确认，支付流水直接成功。
- 拉卡拉动态码返回二维码内容。
- 拉卡拉聚合收银台返回收银台地址或订单信息。
- 拉卡拉支付最终状态以回调或查询结果推进。

## 履约规则

订单支付成功后后端自动履约。

不同订单类型的履约动作：

| 订单类型 | 履约动作 |
| --- | --- |
| 会员充值 | 会员卡余额入账、游戏币入账。 |
| 消费 | 第一阶段推进订单履约状态；具体游戏启动由终端业务控制。 |
| 游戏豆充值 | 商家总账户或门店游戏豆账户入账。 |

产品需要明确：

- 自动履约失败后的后台重试入口和权限。
- 履约失败是否需要提醒运营/门店。
- 消费订单履约成功是否等同于“允许启动游戏”，还是终端需要额外校验设备/场次。

## 退款规则

退款申请支持两种方式：

- 按商品退款：传 `items`，每个明细包含 `orderItemId` 和 `refundQuantity`。
- 指定金额退款：不传明细，传 `refundAmount`。

基础校验：

- 原订单必须存在且属于当前商家。
- 原订单必须已有支付金额。
- 退款金额必须大于 0。
- 同一订单累计活跃退款金额不能超过已支付金额。
- 按商品退款时，累计退款数量不能超过订单明细原购买数量。

退款路径：

- 订单未结算且存在足额可退拉卡拉支付流水：发起拉卡拉原路退款，退款状态为退款中。
- 订单未结算且存在足额可退会员卡内部支付流水：执行会员卡余额或游戏币内部退回。
- 订单已结算、现金支付、混合支付但单一路由不足额等场景：登记为线下退款待处理。

产品需要重点确认：

- 线下退款的运营后台流程：审核、打款登记、凭证上传、完成/驳回。
- 混合支付退款策略：当前实现倾向“一张退款单落在单一路由”，不能拆分时走线下待处理；如果产品要求一张退款单拆分到多条支付流水，需要单独设计拆分退款单。
- 已结算订单退款是否需要生成结算冲减或下期抵扣，目前第一阶段按线下退款处理。

## 结算规则

结算类型：

| 编码 | 类型 |
| --- | --- |
| 1 | 商家结算 |
| 2 | 代理商分润 |

生成结算单参数：

- `settlementType`：商家结算或代理商分润。
- `periodStart`：结算开始日期。
- `periodEnd`：结算结束日期。

系统内部按左闭右开时间段处理：`periodStart 00:00:00` 到 `periodEnd + 1 天 00:00:00`。

商家结算口径：

- 只统计拉卡拉通道成功支付流水净额。
- 结算周期按支付流水成功时间 `pay_time` 计算，不按订单创建时间。
- 净额公式：`支付流水金额 - 支付流水已退金额`。
- 现金、会员卡余额、会员卡游戏币不进入平台拉卡拉钱包，不参与拉卡拉商家结算。
- 结算单生成后，纳入结算的订单从未结算推进为结算中。
- 提现成功回调后，订单推进为已结算。

代理商分润口径：

- 平台利润基数：`累计游戏消耗游戏豆 - 累计游戏退回游戏豆`。
- 代理商收益：`平台利润基数 × 代理商分润比例`。
- 分润比例按小数比例保存，例如 `0.60` 表示 `60%`。
- 代理商分润来自游戏豆账单，不改变订单级结算状态。

提现手续费口径：

- 商家和代理商各有自己的提现手续费率。
- 手续费率按小数比例保存，例如 `0.60` 表示 `60%`。
- 结算配置提供最低结算门槛、最低提现手续费、最高提现手续费。
- 手续费不能超过本次可结算金额。

打款流程：

1. 运营或任务生成结算单，状态为待打款。
2. 发起打款后，状态进入打款中。
3. 平台余额分账给商家或代理商接收方。
4. 分账成功后，通过账管家提现到接收方银行账户。
5. 提现成功回调后，结算单状态为已打款。
6. 调用失败或回调失败时，结算单状态为打款失败，需要后台重试或人工处理。

产品需要补充：

- 结算单审核流程：是否需要人工审核后才能打款。
- 打款失败后的重试规则：自动重试次数、人工重试权限、失败原因展示。
- 结算单关闭规则：什么情况下允许关闭。

## 后台配置需求

产品 PRD 建议补充以下后台配置页面或配置能力：

- 结算配置：结算类型、周期、结算日、最低结算金额、最低提现手续费、最高提现手续费。
- 商家配置：商家手续费率、拉卡拉商家账户配置、分账接收方配置。
- 代理商配置：代理商分润比例、代理商手续费率、拉卡拉代理商账户配置。
- 平台拉卡拉账户配置：平台收款商户号、终端号、机构号等。
- 退款处理台：线下退款待处理列表、退款失败列表、退款完成登记。
- 结算处理台：待打款、打款中、打款失败、已打款列表。
- 拉卡拉请求日志查询：按订单号、退款单号、结算单号、接口类型查看请求/响应/错误摘要。

## 拉卡拉配置参数清单

以下参数需要测试环境和正式环境分别提前提供。测试环境和正式环境不得混用证书、商户号、终端号和回调地址。

### 一、SDK 应用级配置

这些参数用于初始化拉卡拉 SDK，对应 `lakala.sdk.*` 配置：

| 参数 | 用途 | 测试环境 | 正式环境 |
| --- | --- | --- | --- |
| `app-id` | 拉卡拉开放平台应用 ID | 必填 | 必填 |
| `serial-no` | 商户证书序列号 | 必填 | 必填 |
| `private-key-file` | 商户私钥文件，用于请求签名 | 必填 | 必填 |
| `lkl-cer-file` | 拉卡拉公钥证书，用于响应验签 | 必填 | 必填 |
| `lkl-notify-cer-file` | 拉卡拉通知证书，用于异步回调验签 | 必填 | 必填 |
| `sm4-key` | SM4 加密密钥，如拉卡拉要求密文传输则必填 | 按环境要求 | 按环境要求 |
| `server-url` | 拉卡拉 API 网关地址 | 必填 | 必填 |
| `connect-timeout/read-timeout/socket-timeout` | SDK 请求超时配置 | 建议提供 | 建议提供 |

### 二、平台账户配置

平台账户固定 `accountOwnerType=PLATFORM`，`accountOwnerId=0`。用于统一收款、退款和作为余额分账资金来源。

| 参数 | 用途 | 是否必填 |
| --- | --- | --- |
| `merchant_no` | 平台拉卡拉商户号，支付、退款、分账请求使用 | 必填 |
| `term_no` | 平台拉卡拉终端号，主扫、被扫、收银台、退款使用 | 必填 |
| `org_no` | 拉卡拉机构号，余额分账相关请求使用 | 必填 |
| `vpos_id` | 聚合收银台虚拟终端号 | 如聚合收银台要求则必填 |
| `channel_id` | 聚合收银台或渠道参数 | 如拉卡拉接口要求则必填 |
| `status` | 是否启用 | 必填 |

### 三、商家账户配置

每个需要结算打款的商家都需要配置。

| 参数 | 用途 | 是否必填 |
| --- | --- | --- |

### 五、回调地址配置

需要在拉卡拉开放平台分别配置测试环境和正式环境公网可访问地址：

| 回调类型 | 地址 |
| --- | --- |

| 退款回调 | `{baseUrl}/api/trade/lakala/callback/refund` |

- 拉卡拉是否要求 IP 白名单。

### 六、业务费率和结算参数

测试环境和正式环境也需要准备以下业务参数：

- 商家提现手续费率：小数比例口径，`0.60` 表示 `60%`。
- 代理商提现手续费率：小数比例口径，`0.60` 表示 `60%`。
- 代理商分润比例：小数比例口径，`0.60` 表示 `60%`。
- 商家结算配置：结算周期、结算日、最低结算金额、最低提现手续费、最高提现手续费。
- 代理商结算配置：结算周期、结算日、最低结算金额、最低提现手续费、最高提现手续费。

### 七、需要产品提前确认的问题

- 拉卡拉聚合收银台是否必须传 `vpos_id`、`channel_id`，当前代码未强制使用这两个字段。
- 账管家提现是否需要 `bankId` 或银行卡信息参与每次提现请求，当前代码主要使用 `ewallet_account_no` 作为 `payNo`。
- 商家和代理商的银行卡绑定、实名、开户流程由哪个模块维护。
- 混合支付退款是否允许一笔退款单拆分多通道退款。
